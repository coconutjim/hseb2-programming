instantreality 1.0

instantplayer - Test Set

This is a very simple test-set for InstantReality features. Some features which may behave different on various OS and GPU platforms.

They are definitely no examples or tutorials.
Example/Tutorials can be found on the www.instantreality.org site.

If you install the instantplayer on a new machine could you please run this tests and report back any misbehave.

test_anchor.x3d

Simple scene with three anchors: Loading a new X3D file, switching to a viewpoint and opening a webpage in the system's default browser.

test_ecmaScript.x3d

Simple test if the ecma-script interpreter works.

test_java.x3d

Simple test if the java script code works.
You have to have a working java run-time-enviroment.

test_image.x3d

Simple test if the image loader works.

test_sound.x3d

Simple test if the sound loader and render works.

test_glslShader.x3d

Simple teest if the glsl shader works.

test_shadows.x3d

Simple test if the shadow shader/maps works.
You can click on the Model to change the shaderMode.

test_videoInput.x3d

Simple test if the video input from a webcam works.
This is fundamental for the later AR example and application.

test_renderedTexture.x3d

Simple test to check RenderedTexture (using PBuffer or FBO's).


test_dynamicCubeTexture.x3d

Simple test to check the dynamic cube-map update.


test_volumeTexture.x3d

Simple test to check 3D Texture hardware and Image loader.


test_multithread.x3d

Simple test if the parallel application and rendering works.
This should be smooth on most machines and fly on multicores.

test_cg.x3d

Simple test for Cg support.
This should display three lit spheres with textures. The left sphere tests for basic Cg support, the one in the middle for basic CgFX support, and the right one for advanced features of Cg 2.1. CgFX is only supported on Windows and MacOS.

test_markertracker.x3d

Simple test for the VisionLib plugin
This test tries first to load the VisionLib-plugin in InstantIO and afterwards runs the MakerTracker on already available images. No camera needed. Result ok then coordsys model is overlayed and moving with the marker.

test_postertracker.x3d

Simple test for the VisionLib plugin
This test tries first to load the VisionLib-plugin in InstantIO and afterwards runs the PosterTracker on already available images. No camera needed. Result ok then frankfurt skyline looks out of the image sequence.
